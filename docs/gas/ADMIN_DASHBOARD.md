# 管理ダッシュボード（DX・スプレッドシート共有者向け）

利用者向け To-Do と**同じ Web アプリ URL** から、`page` パラメータだけ変えて管理者用画面を開きます。

### まず押さえること（処方的）

| 疑問 | 答え |
|------|------|
| **管理画面だけの「別デプロイ」を新規作成する？** | **不要です。** Web アプリは **1 本のデプロイ**のまま。`admin.html` を追加したら **既存デプロイを「新バージョン」で更新**するだけ。 |
| **デプロイ画面のどこに `ADMIN_DASHBOARD_EMAILS` と書く？** | **書きません。** それは下記の **スクリプト プロパティ**（プロジェクト設定の別画面）にだけ登録します。 |
| **種類は Web アプリで合う？** | **合っています。** ライブラリや API ではなく、**ウェブアプリ**のデプロイでよいです。 |
| **「組織内の全員」にアクセス可にすると、管理画面も全員に見える？** | URL は開けますが、**コードがメールを照合**するので、許可リストに無い人は「権限がありません」になります。 |

### `ADMIN_DASHBOARD_EMAILS` とは（デプロイとは別物）

- **場所**: GAS エディタ **プロジェクトの設定（歯車）→ スクリプト プロパティ** で **新規追加**するキーと値。  
- **キー（名前）**: そのまま `ADMIN_DASHBOARD_EMAILS`  
- **値**: `r-kusaka@okamoto-group.co.jp,colleague@okamoto-group.co.jp` のように **カンマ区切り**（DX メンバーなど、管理画面を見てよい人のメール）。  
- **役割**: サーバー側が「今ログインしている Google アカウントのメールが、この一覧に含まれるか」を確認し、**`?page=admin` の画面だけ**通す／弾く。  
- **未設定**: スプレッドシートの**所有者**だけが管理画面を開ける想定。

## 閲覧までの流れ（運用）

1. **GAS プロジェクトにファイルを追加**  
   - エディタで **「＋」→「HTML」** から `admin` を追加し、`docs/gas/admin.html` の内容を貼り付けて保存する。  
   - （ファイル名は `admin` でよい。拡張子 `.html` はエディタが付ける。）

2. **`Code.gs` を最新版に**  
   - リポジトリの `docs/gas/Code.gs` を丸ごとコピーしてデプロイ済みプロジェクトに反映する。  
   - `getAdminDashboardData` / `sendAdminTaskReminder` および受信者一覧用のヘルパーが含まれていることを確認する。

3. **Web アプリを再デプロイ**  
   - **デプロイ → デプロイを管理 → 既存のデプロイを編集（鉛筆）→ バージョン「新バージョン」→ デプロイ**  
   - 利用者向け `index` と同じデプロイでよい（別デプロイは不要）。

4. **管理者メールの設定（推奨）**  
   - GAS エディタ: **プロジェクトの設定（歯車）→ スクリプト プロパティ**  
   - プロパティ: `ADMIN_DASHBOARD_EMAILS`  
   - 値: `dx1@example.com,dx2@example.com` のように**カンマ区切り**（来期 DX メンバーなど）。  
   - **未設定の場合**は、**スプレッドシートの所有者アカウントだけ**がダッシュボードを開ける想定になる。

5. **開き方（ブックマーク用 URL）**  
   - 利用者アプリの URL が  
     `https://script.google.com/macros/s/XXXX/exec`  
     のような形なら、管理画面は  
     `https://script.google.com/macros/s/XXXX/exec?page=admin`  
   - **同じ Google アカウントでログインしたブラウザ**で開く（`Session.getActiveUser()` が使えるため）。

6. **権限がない場合**  
   - 「閲覧する権限がありません」と出たら、  
     - スクリプトプロパティの `ADMIN_DASHBOARD_EMAILS` に自分のメールを追加するか、  
     - スプレッドシートを自分のドライブにコピーして所有者になる（非推奨・運用は要相談）。

## 画面でできること（2025 改訂 UI）

見た目は利用者向け To-Do と同系（背景 `#f2f2f7`、白カード、アクセントカラー）。**アクセント色**は利用者アプリと同じ `localStorage` キー `taskmaster_accent_id` を共有するため、どちらで変えても揃います。

### タブ（左上）

| タブ | 内容 |
|------|------|
| **社員依頼** | 社員向け依頼の一覧・進捗 |
| **店舗依頼** | 店舗向け依頼の一覧・進捗 |
| **定期配信** | 「定期配信データ」シートの登録一覧（参照のみ） |

### 依頼一覧（社員・店舗）

- **集計チップ**: 件数 / 未完了 / 期限超過 / 完了  
- **フィルタ**: すべて / 未完了のみ / 期限超過のみ  
- **各行**: ステータス・種別・種類・内容・期限・送信者・進捗バー  
- **行をタップして展開** → 未実施者（社員）または未完了店舗（店舗）をチェック  
- **「選択した人にリマインド」** → 「○○ 名にリマインドを送信しますか？」確認後、一括メール送信  
  - 社員依頼: 未実施の社員メールへ  
  - 店舗依頼: 未完了店舗を選ぶと、その店舗の担当社員へ（重複メールは1通にまとめる）

### ヘッダー

- **再読み込み** / **色**（アクセント10色） / **スプレッドシート**（権限がある場合） / **利用者アプリ**

## サーバー側 API

| 関数 | 用途 |
|------|------|
| `getAdminDashboardData()` | 一覧・集計・`employeeTasks` / `storeTasks` / `scheduledTasks` と各 task の `recipients` |
| `sendAdminTaskReminder(taskId, keys, mode)` | 依頼1件・選択した未実施者／店舗担当へリマインドメール（`mode`: `'employee'` \| `'store'`） |

（従来のユーザー別リマインド `sendAdminReminderEmail` が残っている場合は、新 UI では上記の依頼単位リマインドを主に使います。）

## セキュリティの整理

- 一般社員に**スプレッドシート本体を共有しない**運用と組み合わせる。  
- Web アプリの「アクセスできるユーザー」は、利用者アプリと同じ設定（例: 組織内のみ）にするとよい。  
- 管理 URL（`?page=admin`）は**知っている人だけが開ける URL** ではなく、**メールホワイトリストでサーバー側チェック**している点に注意。

## 期限リマインド自動送信（2日前・前日・当日 8:00）

`admin.html` の手動リマインドと同じ未実施判定を使い、**社員依頼・店舗依頼・TFチーム依頼**すべてを対象に、期限の **2日前・前日・当日** の朝 **8:00（JST）** に未実施者へだけ自動送信します。期限超過後は送りません（最大3回で終了）。

### セットアップ手順

1. **`Code.gs` を最新版に反映**（`processDeadlineRemindersBatch` 等が含まれること）
2. **完了レポート**は既定で `r-kusaka@okamoto-group.co.jp` のみ（`ADMIN_DASHBOARD_EMAILS` には送らない）  
   - 変更する場合のみスクリプトプロパティ `DEADLINE_REMINDER_REPORT_EMAIL` を設定
3. GAS エディタで **`setupDeadlineReminderTrigger`** を1回実行 → 毎日 8:00 の時間主導トリガーが登録される
4. 初回実行後、スプレッドシートに **`リマインド送信履歴`** シートが自動作成される（重複送信防止用）

### 完了レポートメール

バッチ実行後、**送信があった日**（または対象タスクがあった日・エラー時）に **日下のみ**へ **「リマインドメール送信完了のお知らせ」** が届きます。

- 送信件数
- 送信先メールアドレス一覧（縦並び）
- 対象タスクの概要
- エラーがあれば詳細

自動リマインドの **Reply-To は付けない**（From は実行アカウント＝日下）。  
手動リマインド（admin 画面）は従来どおり。自動送信とは別枠で動作します。

### サーバー側 API（追加）

| 関数 | 用途 |
|------|------|
| `processDeadlineRemindersBatch()` | 時間主導トリガーから呼ばれる本体 |
| `setupDeadlineReminderTrigger()` | 毎日 8:00 トリガーの初回登録 |
