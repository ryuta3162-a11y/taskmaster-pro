# To Do List 動画台本 v2（全章・収録用）

現行アプリ・ガイドサイト（https://todo-list-guide.vercel.app/）の内容に合わせた **2026年版** です。  
**画面収録は無音**で行い、ナレーションは `guide-site/scripts/generate_narration.py` で自動生成した音声を後から合成します。

---

## 制作方針

| 項目 | 方針 |
|------|------|
| 収録 | 画面操作のみ（マイクは使わない） |
| ナレーション | 自動音声（edge-tts）→ MP3 を動画に合成 |
| 字幕 | ナレーション生成時の VTT をそのまま利用可 |
| テロップ | 各章の **【テロップ】** を画面録画時にメモ帳 or 編集ソフトで重ねても可 |
| 1章の長さ | おすすめ 1.5〜4分（長くても5分以内） |

### 動画本数（推奨 7本）

| ファイル名 | 章 | 目安 | 優先度 |
|------------|-----|------|--------|
| `overview.mp4` | 全体の流れ | 3分 | ★★★ |
| `register.mp4` | ログイン・初回登録 | 2分 | ★★ |
| `checklist.mp4` | リストチェック | 4分 | ★★★ |
| `request.mp4` | 新規投稿 | 4分 | ★★★ |
| `repost.mp4` | 再投稿 | 2分 | ★★ |
| `scheduled.mp4` | 定期配信 | 3分 | ★★ |
| `progress.mp4` | TF進捗ダッシュボード | 2分 | ★★ |

※ Gmailラベル・通知メール・ログアウトは **checklist / register 内で触れる** か、テキストガイドで補完。

## 0. 全体の流れ（overview・約3分）【新規】

**対象** 全員（最初に1本だけ見せる）

**画面の流れ**  
ホーム全体 → 4つのメニューを順にハイライト → リストチェックを軽く表示 → チーム進捗ボタンを表示

**ナレーション台本**

> To Do List の使い方を、かんたんにご紹介します。  
> ホーム画面には、大きく4つのメニューがあります。  
> **新規投稿** で依頼を出す。**再投稿** で過去の依頼をもう一度送る。**定期配信** で毎月自動で送る。**リストチェック** で、届いた To Do を完了する。  
> 依頼を受け取る方は、**リストチェック** だけ覚えれば大丈夫です。  
> 依頼を出す方は、**新規投稿** を中心に使います。  
> チームの進み具合を見る方は、ホーム下の **チーム進捗管理ダッシュボード** を開きます。  
> 詳しい操作は、目的に合った動画を見てください。  
> まずは自分の役割に合う章から始めましょう。

**【テロップ】** 送る＝新規投稿・再投稿・定期配信 ／ 受ける＝リストチェック ／ 見る＝進捗ダッシュボード

**撮影メモ**
- [ ] ホームの未完了バッジが見える状態
- [ ] 4ボタンをカーソルで順に指す（クリックしなくても可）
- [ ] 実送信はしない

## 1. ログイン・初回登録（register・約2分）

**画面の流れ**  
ログイン → プロフィール登録（店舗エリア所属パターン）→ 保存 → （任意）EAST本部タブを切り替えて表示

**ナレーション台本**

> 初めての方は、社内メールアドレスでログインし、プロフィールを登録します。  
> **名前・役職・チーム** は必ず選んでください。配信先の判定に使われます。  
> **店舗エリアに所属する方** は、エリアとテリトリーを選び、**管轄店舗** を登録します。  
> **EAST本部に所属する方** は、エリアの代わりに **EAST本部** を選べば、店舗の選択は不要です。  
> 登録が終わったら **変更を保存** を押します。  
> あとから直すときは、ログイン画面で **ログイン情報を変更** から同じ項目を編集できます。

**【テロップ】** チーム・役職は必須 ／ EAST本部は店舗選択不要

**撮影メモ**
- [ ] 店舗エリア所属の流れをメインで収録
- [ ] EAST本部タブは5秒程度の切り替えでOK
- [ ] 実在メールは映さない（テスト用 or 自分のアカウント）

## 2. リストチェック（checklist・約4分）【最重要】

**画面の流れ**  
ホーム → リストチェック → 未実施タブ → 種別フィルタ → 店舗チップ（店舗依頼がある場合）→ 完了操作 → 実施済みタブ → 取り消し → 添付（ZIP・画像）を開く

**ナレーション台本**

> 依頼を受け取った方は、ホームの **リストチェック** を開きます。  
> 右上の数字は、**未完了の件数** です。  
> **未実施** タブに、対応が必要な To Do が並びます。  
> 上部では **社員依頼・店舗依頼・TFチーム依頼** で絞り込めます。  
> **店舗依頼** の場合は、自分の担当店舗ごとに完了します。店舗名のチップで絞り込んでから、完了を押してください。  
> 内容を確認し、対応が終わったら **完了** を押します。  
> リンクや **ZIP・画像の添付** がある場合は、ボタンから開けます。  
> 間違えて完了した場合は、**実施済み** タブから **取り消し** できます。  
> 配信が完了すると **通知メール** も届きます。件名は、【TODOリスト】新しいタスクが届きました、です。  
> 依頼を出さない方は、リストチェック専用の URL をブックマークしておくと便利です。ガイドページに URL を掲載しています。

**【テロップ】** 受け手はこの画面だけ覚えればOK ／ 店舗依頼は店舗ごとに完了

**撮影メモ**
- [ ] テスト用タスクで完了→取り消しまで実演
- [ ] ZIP付きタスクがあれば「ZIPを開く」を表示
- [ ] ブックマークURLはガイドのコードチップを画面端に小さく映す程度で可

## 3. 新規投稿（request・約4分）

**画面の流れ**  
ホーム → 新規投稿 → 3種類の依頼を切り替え → 内容・期限入力 → 添付（画像・ZIP）→ 配信先確認 → 送信（テスト用・自分宛で可）

**ナレーション台本**

> ホームの **新規投稿** から、依頼を作成します。  
> まず **依頼の種類** を選びます。  
> **社員への依頼** は、役職別に配信します。  
> **店舗への依頼** は、エリアや店舗を選び、担当店舗ごとに完了してもらいます。  
> **TFチームの依頼** は、所属チーム別に配信します。チームのメンバーそれぞれが完了する必要があります。  
> 次に **依頼内容** と **期限** を入力します。  
> 参考資料があれば、**画像・PDF・ZIP** を添付できます。ZIPは最大25メガバイトまでです。  
> 右側で **誰に届くか** を確認し、配信しない人だけ外すこともできます。  
> 問題なければ **送信** を押します。対象者にメールが届き、リストチェックに To Do が表示されます。

**【テロップ】** 社員＝役職 ／ 店舗＝店舗単位完了 ／ TF＝チームメンバー各自完了

**撮影メモ**
- [ ] 3種類のタブを順に表示（実送信は1回で可）
- [ ] ZIP添付UI「ZIPを追加」を映す
- [ ] 本番全体配信は避け、自分宛テスト推奨

---

## 4. 再投稿（repost・約2分）

**画面の流れ**  
ホーム → 再投稿 → 一覧 → 再利用して作成 → 期限変更 → 送信

**ナレーション台本**

> **再投稿** では、過去に **新規投稿** で送った依頼だけが一覧に出ます。  
> **定期配信** で登録したものは、ここには表示されません。  
> もう一度送りたい依頼の **再利用して作成** を押すと、内容が引き継がれます。  
> 依頼内容・URL・添付・配信先はそのまま。**期限だけは毎回選び直してください。**  
> 確認して **送信** すれば、新規投稿と同じように配信されます。  
> 毎月自動で送りたい場合は **定期配信** を使います。

**撮影メモ**
- [ ] 定期配信一覧と混同しないよう、再投稿画面であることを冒頭で映す

## 5. 定期配信（scheduled・約3分）

**画面の流れ**  
ホーム → 定期配信 → 新規フォーム → 配信日・期限の設定 → 稼働中一覧 → 編集

**ナレーション台本**

> **定期配信** は、毎月決まった日に、同じ To Do を **自動で送る** 設定です。  
> 新規投稿と同様に、依頼の種類・内容・配信先を設定します。  
> **毎月何日に配信するか** を選びます。配信時刻は午前10時固定です。  
> 毎月のタスク期限は、**月末** または **○日まで** から選べます。  
> **今月の初回分は作成しない** にチェックすると、翌月の配信日から始められます。  
> 登録後は **稼働中の定期配信** 一覧から、編集や停止ができます。編集は **次回の配信から** 反映されます。

**撮影メモ**
- [ ] 再投稿との違いがわかるよう、定期配信画面のタイトルを映す

## 6. TFチーム進捗管理ダッシュボード（progress・約2分）【新規】

**画面の流れ**  
ホーム → チーム進捗ボタン → ダッシュボード → 進捗率・完了者一覧

**ナレーション台本**

> ホーム画面の下にある **チーム進捗管理ダッシュボード** は、進捗を **見るだけ** の画面です。  
> **TFチームの依頼** など、チームに関係する To Do の進み具合を確認できます。  
> 依頼ごとに、進捗率・期限・誰が完了したかが一覧でわかります。  
> 自分の To Do を完了する操作は、**リストチェック** で行います。ここでは完了ボタンはありません。  
> 所属チームが複数ある方は、チームごとにボタンが表示されます。

**【テロップ】** 見る専用 ／ 完了はリストチェック

**撮影メモ**
- [ ] TF依頼が1件以上あるチームで開く
- [ ] 進捗バー・完了人数が見える状態

## 付録A. Gmailラベル（gmail-label・約2分）【任意・単独動画可】

**画面の流れ**  
Gmail → フィルタ作成 → 件名条件 → To-Do List ラベル付与

**ナレーション台本**

> To Do の通知メールは、件名が【TODOリスト】新しいタスクが届きました、で統一されています。  
> Gmail のフィルタで **To-Do List** ラベルを自動で付けると、受信トレイが整理しやすくなります。  
> 検索ボックスに、件名の条件を入力します。  
> フィルタを作成し、**ラベルを付ける** で To-Do List を選びます。  
> ラベルがなければ新規作成してください。  
> 設定後は、To Do の通知だけを左メニューから一覧できます。

※ ガイドサイトの手順と同じ。動画化しなくてもテキストで十分な章。
