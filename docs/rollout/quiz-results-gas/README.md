# 理解度チェック集計 GAS

To Do List 本体とは分けた、ガイドサイト専用の集計コードです。

## スプレッドシート

- ID: `1NvJrgfanwN8XMu9YQh5tFbrqDHxU7fuJYJteLzfKqbI`
- シート名: `テスト`
- 列構成:
  - A: 名前
  - B: メールアドレス
  - C-H: 1問目-6問目
  - I: 合格/不合格

## 使い方

1. 集計用スプレッドシートを開く
2. 拡張機能 → Apps Script
3. `Code.gs` にこのフォルダの `Code.gs` を入れる
4. デプロイ → 新しいデプロイ → 種類は「ウェブアプリ」
5. 実行ユーザーは自分、アクセス権は運用に合わせて設定
6. 発行されたウェブアプリ URL を `docs/rollout/guide-site/guide-config.js` の `quizResultEndpoint` に入れる

既存のウェブアプリを更新する場合は、`Code.gs` を差し替えたあとに **デプロイを管理 → 編集 → 新バージョン** で更新してください。コードを保存しただけでは公開URL側へ反映されません。

## 動き

ガイドサイトからメールアドレスと6問分の回答を受け取り、B列のメールアドレスが一致する行だけを更新します。メールアドレスが見つからない場合は書き込みません。

各問題は「① 依頼の種類」と「② 配信方法」の2つを回答し、両方正解した場合だけその問題を `○` として記録します。回答の再送信時も、同じ列（C〜I列）を上書き更新します。

`action=verify` でメールアドレスの存在確認、`action=submit` でC-H列とI列の記録を行います。ガイドサイト本体は開始を速くするため、最初の画面では通信せず、送信時にB列との一致を確認します。
