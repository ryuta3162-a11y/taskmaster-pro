# 説明会メール：Gmail 下書き保存 & カレンダー登録

## Gmail の下書きに保存する（HTML の見た目を保つ）

**HTML ファイルは Gmail に自動連携されません。** 次の手順で「下書き」にします。

1. `testrun-meet-invitation.html` を **Chrome / Edge** で開く（ダブルクリック）
2. `Ctrl + A` → `Ctrl + C` で全体コピー
3. [Gmail Web版](https://mail.google.com) → **作成**
4. **件名**を入力（例：`【ご案内】新 To-Do List テストラン説明会`）
5. **本文**をクリック → `Ctrl + V` で貼り付け
6. 見た目を確認
7. **送信せずに** ウィンドウ右上の **×** を押す → 「下書きに保存」で **OK**

→ 左メニューの **下書き** に、同じデザインのメールが残ります。あとから開いて編集・送信できます。

### 何度も使う場合（テンプレート）

1. 上と同じく貼り付けた状態で、作成画面右下 **︙** → **テンプレート** → **下書きを保存**
2. 次回：**︙** → **テンプレート** → 保存した名前を選ぶ

※ スマホアプリより **PC の Gmail Web版** の方がレイアウトが崩れにくいです。

## 「この日程で参加する」＝ Google カレンダーに追加

メールのボタンは **Google カレンダーの「予定を作成」画面** を開くリンクです。

1. 受信者がボタンをクリック
2. カレンダーに日時・タイトル・Meet URL が **あらかじめ入った状態** で開く
3. 受信者が **「保存」** を押す → **自分のカレンダー** に登録される

**注意:** セキュリティのため、クリックだけで勝手にカレンダーに入ることはできません（必ず本人が「保存」を押す）。これが一般的なやり方です。

### 日程・Meet を変えるとき

`testrun-meet-invitation.html` 内の **【要編集】** を直します。

| 直す場所 | 内容 |
|----------|------|
| 表示の日時 | `6月○日（○） 10:00〜11:00` など |
| ボタンの `href`（カレンダーURL） | `dates=` の開始・終了（下記） |
| `details=` の中 | Meet の URL |
| Meet の文字リンク | `【要編集】MeetリンクA` など |

### カレンダー URL の `dates=` の書き方（日本時間）

`ctz=Asia/Tokyo` があるので、**日本時間の数字** をそのまま書きます。

| 例（日程A） | 値 |
|-------------|-----|
| 2026年6月10日 10:00〜11:00 | `dates=20260610T100000/20260610T110000` |

形式: `YYYYMMDD` + `T` + `HHMMSS`（24時間）

### カレンダー URL を自動生成する（Google公式）

1. [Google カレンダー](https://calendar.google.com) で該当日時の予定を **新規作成**
2. タイトル・説明（Meet URL）・日時を入力
3. 右上 **︙** → **公開する** などではなく、ブラウザの予定詳細 URL を使うより、  
   **「カレンダーに追加」リンク生成ツール** として次を使うと楽です:  
   https://github.com/InteractionDesignFoundation/add-event-to-calendar-docs/blob/main/services/google.md

簡易手順: 予定を一度作って「ゲストにメール送信」ではなく、HTML の `href` を [この形式](https://calendar.google.com/calendar/render?action=TEMPLATE&text=テスト&dates=20260610T100000/20260610T110000&ctz=Asia/Tokyo) で組み立てる。
