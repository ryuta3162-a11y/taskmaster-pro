# GAS で説明会メールの下書きを作る

**結論:** Gmail 下書きの自動作成は **GAS の方がずっと楽** です。  
カレンダー登録は GAS でも「相手のカレンダーに勝手に入れる」ことはできません（Google の仕様）。メール内のボタン → カレンダー画面 → 本人が「保存」、という形は同じです。

## 何が実現できるか

| やりたいこと | GAS でできる？ |
|--------------|----------------|
| HTML メールを **Gmail の下書き** に入れる | **◎ できる**（`createMeetInvitationDraft`） |
| 送ったときと同じ見た目 | **◎ できる**（`htmlBody` で渡す） |
| 希望者だけを **1つの予定のゲスト** にする | **◎ この運用**（メールは参加希望 → 主催がゲスト追加） |
| ボタンで各自が新規予定を作る（複製っぽくなる） | **× やめた**（旧方式） |

## セットアップ（5分）

### 1. GAS に貼る

**ToDo 用スプレッドシート** → **拡張機能** → **Apps Script**

- 新規ファイル `MeetInvitationMail.gs` を追加
- `docs/rollout/gas-invitation-email/MeetInvitationMail.gs` の中身を **丸ごとコピペ**
- 保存

（既存の `Code.gs` とは別ファイルで OK）

### 2. 日程を編集

`MeetInvitationMail.gs` の **`INVITE_CONFIG`** だけ直す:

- `label` … 表示用の日時
- `start` / `end` … `new Date(2026, 5, 10, 10, 0, 0)`（月は 0=1月）
- `meetUrl` … 各回の Meet リンク
- `signatureContact` … 署名

### 3. 実行

1. 関数一覧で **`createMeetInvitationDraft`** を選ぶ  
2. **実行** ▶  
3. 初回は **Gmail の権限** を許可  
4. 「下書きを作成しました」と出たら **Gmail → 下書き** を開く  
5. **宛先（To）** を入れて送信  

### 4.（任意）メニューから実行

スプレッドシートに戻り、リロードすると **「案内メール」→「説明会メールを下書き作成」** が出ます。  
※ 既存の `onOpen` と名前がぶつかる場合は、手動実行だけ使ってください。

## コピペ方式との違い

| | ブラウザで HTML をコピー | **GAS（このスクリプト）** |
|--|-------------------------|---------------------------|
| 下書き作成 | 手動で貼り付け | **実行1回で下書き完成** |
| 日程・Meet URL | HTML 内を3か所ずつ編集 | **`INVITE_CONFIG` だけ** |
| カレンダー URL | 自分で組み立て | **自動生成** |

## 希望者だけゲストにする（主催者の作業）

1. 案内メールを送る（ボタンは **参加希望** → 日下さんにメールが届く）
2. 参加希望のメールが来た人だけ、Google カレンダーで該当予定（日程A〜D）を開く
3. **ゲスト** タブ → **ゲストを追加** にメールアドレスを入れる → 保存
4. 相手に **カレンダー招待** が届く → 相手が承諾すると、**同じ1件の予定** に載る（複製しない）

スクリーンショットの「ゲストを追加」が、この理想の入口です。
